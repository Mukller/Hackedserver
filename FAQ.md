# Часто задаваемые вопросы (FAQ) — HackedServer

Ответы на популярные вопросы о HackedServer.

---

## 📍 Общие вопросы

### Q: Что такое HackedServer?

**A:** HackedServer — это премиум-плагин для Minecraft серверов, который защищает от читов и неавторизованных модификаций через анализ сетевых пакетов. Он автоматически обнаруживает и блокирует различные типы читов в реальном времени.

### Q: Сколько стоит HackedServer?

**A:** HackedServer — это закрытый проект. Информацию о лицензировании уточняйте у разработчика [@Mukller](https://github.com/Mukller).

### Q: На каких версиях Minecraft работает?

**A:** HackedServer поддерживает Minecraft версии:
- ✅ 1.8.x - 1.12.x
- ✅ 1.13.x - 1.16.x (рекомендуется)
- ✅ 1.17.x - 1.21.x (последние)

Для лучшей совместимости используйте версии 1.20.x или выше.

### Q: Какой server software поддерживается?

**A:** Поддерживаются:
- ✅ **Paper** — рекомендуется
- ✅ **Spigot**
- ✅ **Purpur**
- ✅ **CraftBukkit**
- ❌ Fabric, Forge — не поддерживаются

Paper рекомендуется использовать, так как он имеет лучшую оптимизацию.

## 🚀 Установка и запуск

### Q: Как установить HackedServer?

**A:** Полная инструкция в [INSTALL.md](INSTALL.md), но кратко:
1. Скачайте `hackedserver-3.16.1.jar`
2. Поместите в папку `plugins/`
3. Перезагрузите сервер
4. Готово!

### Q: Плагин не загружается. Что делать?

**A:** Проверьте:
1. Версию Java: `java -version` (должна быть 8+)
2. Целостность файла: `ls -lh plugins/hackedserver-3.16.1.jar`
3. Логи сервера: `tail logs/latest.log`
4. Переустановите плагин

Если не помогает — создайте [Issue](https://github.com/Mukller/Hackedserver-/issues) на GitHub.

### Q: Нужна ли переустановка при обновлении?

**A:** Нет, просто:
1. Положите новый JAR в папку `plugins/`
2. Выполните `/reload`
3. Старая конфигурация сохранится автоматически

### Q: Конфликтует ли с другими anti-cheat плагинами?

**A:** Может конфликтовать. Если у вас уже установлен другой anti-cheat, рекомендуется:
- Использовать только один плагин
- Или тщательно настроить оба, чтобы они не противоречили друг другу

Обычно достаточно отключить дублирующиеся проверки в одном из плагинов.

## ⚙️ Конфигурация

### Q: Как изменить режим детектирования?

**A:** Отредактируйте файл `plugins/HackedServer/config.yml`:

```yaml
detection-mode: NORMAL  # STRICT, NORMAL или PERMISSIVE
```

Затем выполните: `/hs reload`

### Q: Как отключить определенный тип проверки?

**A:** В конфиге найдите раздел `detectors`:

```yaml
detectors:
  aimbot: false         # отключить Aimbot проверку
  speed-hack: true      # оставить включенным
  fly-hack: true
  killaura: false
```

### Q: Где находятся логи?

**A:** Логи находятся в:
```
plugins/HackedServer/logs/events.log
```

Вы также можете посмотреть их командой:
```
/hs logs
```

### Q: Как добавить игроков в белый список?

**A:** Отредактируйте `plugins/HackedServer/config.yml`:

```yaml
whitelist:
  enabled: true
  players:
    - "AdminName"
    - "ModeratorName"
    - "TrustedPlayer"
```

Затем: `/hs reload`

### Q: Как изменить сообщение при выдаче?

**A:** В конфиге:

```yaml
punishment:
  kick-message: "&c&lВЫ БЫЛИ ВЫКИНУТЫ&r&7 за использование читов!"
  ban-message: "&c&lВЫ ЗАБАНЕНЫ&r&7 на 7 дней за читерство!"
```

Используйте `&` для цветов (например, `&c` = красный).

## 📊 Производительность

### Q: Влияет ли плагин на производительность сервера?

**A:** Влияние минимально:
- Занимает ~50-100 MB памяти
- Потребление CPU < 2% на среднем сервере
- На 200+ игроков может потребоваться дополнительная оптимизация

### Q: Как уменьшить потребление ресурсов?

**A:** Несколько способов:
1. Отключите ненужные проверки в конфигурации
2. Увеличьте интервал между проверками
3. Выберите режим PERMISSIVE вместо STRICT
4. Увеличьте RAM сервера: `java -Xmx4G -jar server.jar nogui`

### Q: Есть ли ложные срабатывания?

**A:** Да, особенно на режиме STRICT. Способы уменьшения:
1. Переключитесь на NORMAL или PERMISSIVE
2. Добавьте проблемных игроков в белый список
3. Отключите чувствительные проверки
4. Сообщите разработчику через Issue

## 🛡️ Безопасность и читы

### Q: Какие читы обнаруживает HackedServer?

**A:** Основные типы:
- ✅ Aimbot и автоприцел
- ✅ Speed Hack
- ✅ Fly Hack
- ✅ Killaura
- ✅ Reach Hack
- ✅ Noclip
- ✅ Anti-knockback
- ✅ Damage Hack

Однако **100% защиту невозможно гарантировать** — продвинутые читы могут остаться незамеченными.

### Q: Может ли плагин быть обойден?

**A:** Теоретически да. Как и любое анти-чит решение, опытные читеры могут найти способы, но это значительно сложнее. Плагин постоянно обновляется для борьбы с новыми методами.

### Q: Что делать, если игрок обошел защиту?

**A:** Сообщите об этом:
1. Создайте [Issue](https://github.com/Mukller/Hackedserver-/issues) с деталями
2. Приложите логи и доказательства
3. Включите видео использования чита
4. Разработчик приложит усилия для исправления

### Q: Защищает ли плагин от модов?

**A:** Частично. Плагин может обнаружить многие неавторизованные моды (Rei's Minimap, Xray и т.д.), но не все. Для полной защиты комбинируйте с другими решениями.

## 💬 Команды и разрешения

### Q: Какие команды доступны?

**A:** Основные команды:

| Команда | Описание | Разрешение |
|---------|---------|-----------|
| `/hs status` | Статус плагина | `hackedserver.status` |
| `/hs reload` | Перезагрузить конфиг | `hackedserver.admin` |
| `/hs check <игрок>` | Проверить игрока | `hackedserver.check` |
| `/hs logs` | Посмотреть логи | `hackedserver.logs` |

### Q: Как дать разрешение игроку?

**A:** Используйте плагин управления разрешениями (LuckPerms):

```
/lp user USERNAME permission set hackedserver.admin true
```

Или в конфиге плагина (зависит от вашей системы разрешений).

### Q: Может ли администратор обойти проверки?

**A:** Если добавить в белый список или дать разрешение `hackedserver.bypass`, то да. Будьте осторожны с этим разрешением.

## 🆘 Проблемы и решения

### Q: "Permission denied" при установке

**A:**
```bash
chmod 644 plugins/hackedserver-3.16.1.jar
chown minecraft:minecraft plugins/hackedserver-3.16.1.jar
```

### Q: "Out of Memory" ошибка

**A:** Увеличьте RAM:
```bash
java -Xmx4G -Xms2G -jar server.jar nogui
```

### Q: Плагин тормозит сервер

**A:**
1. Отключите сложные проверки
2. Увеличьте интервал между проверками
3. Выберите PERMISSIVE режим
4. Перейдите на Paper server (лучше оптимизирован)

### Q: Много ложных срабатываний

**A:**
1. Переключитесь на NORMAL или PERMISSIVE
2. Добавьте игроков в whitelist
3. Отключите чувствительные детекторы
4. Сообщите разработчику
